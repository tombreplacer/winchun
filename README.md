Программа предназначена для преодоления региональных рестрикций, перенаправляя выбранные домены на SOCKS-прокси **без включения глобального туннельного режима** на самом VPN-клиенте. 

**Мотивация создания:** На Linux подобная точечная маршрутизация (policy routing / process-based routing) реализуется достаточно просто с помощью `iptables` и `ip rule`. Однако, хорошего, стабильного и бесплатного аналога для Windows, работающего на уровне системы без костылей, не нашлось. WinChun решает эту проблему элегантно и работает «из коробки».

WinChun работает system-wide: она перехватывает трафик к выбранным сайтам от любых приложений (браузеры, игры, CLI-утилиты) и заворачивает его в ваш SOCKS5, оставляя весь остальной трафик идти напрямую через провайдера.

## Как это работает

```text
[Любое приложение] → [Таблица маршрутов Windows]
    ├── Домены из списка    → [TUN адаптер] → [tun2socks] → [SOCKS5] → интернет
    └── Остальной трафик    → [Основной шлюз] → напрямую
```

1. Программа запускает `tun2socks` и создает виртуальный сетевой адаптер (TUN) `winchun0`.
2. Домены из файла конфигурации разрешаются в IP-адреса.
3. Для этих IP добавляются маршруты через TUN-адаптер в Windows.
4. **Встроенный WFP-фильтр** блокирует возможность образования петли маршрутизации, запрещая прокси-клиенту отправлять трафик обратно в TUN-адаптер.
5. При закрытии (по `Ctrl+C`) все маршруты и WFP-правила корректно удаляются, система остается чистой.

## Требования

- Windows 10/11
- Права администратора (для создания TUN-адаптера и WFP-правил)
- Работающий SOCKS5 прокси (например, Xray, v2rayN, Hiddify и т.д.)

## Установка и запуск

Для работы нужны всего два файла в одной папке: `winchun.exe` и `config.json`.
Библиотека `wintun.dll` и утилита `tun2socks.exe` уже зашиты внутрь с помощью `go:embed`.

### 1. Конфигурация

Создайте или скопируйте `config.json` (в репозитории есть `config.json.example`):

```json
{
  "socks5": "127.0.0.1:10810",
  "tun_name": "winchun0",
  "tun_addr": "10.0.85.2",
  "tun_gw": "10.0.85.1",
  "tun_mask": "255.255.255.0",
  "dns_refresh_seconds": 120,
  "upstream_ips": [],
  "domains": [
    "chatgpt.com",
    "*.ipinfo.io"
  ]
}
```

- Укажите правильный порт вашего `socks5`.
- Добавьте нужные домены в `domains`.
- В `upstream_ips` можно указать IP-адреса ваших VPS-серверов (опционально, для дополнительной защиты от блокировок).

### 2. Как запускать и проверять

1. Запустите ваш прокси-клиент (Xray/v2ray/и т.д.). Убедитесь, что SOCKS5 работает.
   * *Как проверить SOCKS5:* выполните команду в PowerShell:
     `curl -x socks5://127.0.0.1:10810 https://ipinfo.io` (укажите ваш порт). Если возвращается IP вашего VPN, значит прокси работает.
2. Откройте консоль от имени **Администратора**.
3. Запустите `winchun.exe`.
4. Проверьте работу программы на примере `ipinfo.io`:
   * **БЕЗ WinChun** (или если `ipinfo.io` нет в списке): зайдите в браузере на `https://ipinfo.io`. Вы увидите ваш реальный домашний IP-адрес.
   * **С включенным WinChun** (и `*.ipinfo.io` в `config.json`): зайдите на `https://ipinfo.io` еще раз. Вы должны увидеть IP-адрес вашего VPN-сервера.

## Остановка

Нажмите `Ctrl+C` в консоли. Утилита:
1. Удалит все добавленные маршруты.
2. Остановит tun2socks и удалит виртуальный адаптер.
3. Автоматически зачистит WFP-правила.

## Сборка из исходников

1. Установите Go.
2. Запустите файл `build.bat` (он выполнит `go build -o winchun.exe ./src`).
