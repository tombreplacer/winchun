# WinChun

Утилита для селективной маршрутизации трафика через SOCKS5 прокси на Windows.  
Направляет **только трафик к указанным доменам** через SOCKS, остальной трафик идёт напрямую.

Работает **system-wide** — перехватывает трафик всех приложений (браузеры, игры, CLI-утилиты и т.д.).

**Внимание:** Утилита работает в связке с вашим SOCKS5-клиентом (v2rayN, Nekobox, Hiddify и т.д.). В вашем клиенте режим маршрутизации (Routing) должен быть установлен на **Global / Проксировать всё**, иначе возникнет бесконечная петля переадресации!

## Как это работает

```
[Любое приложение] → [Таблица маршрутов Windows]
    ├── IP домена из списка → [TUN адаптер] → [tun2socks] → [SOCKS5] → назначение
    └── Всё остальное       → [Default Gateway] → напрямую
```

1. Утилита извлекает встроенные `tun2socks` и `wintun.dll` во временную папку.
2. Запускается `tun2socks` — создаёт виртуальный сетевой адаптер (TUN) `winchun0`.
3. Домены из файла конфигурации резолвятся в IP-адреса.
4. Для этих IP добавляются маршруты через TUN-адаптер в Windows.
5. DNS периодически обновляется (домены могут менять IP динамически).
6. По `Ctrl+C` — все маршруты корректно удаляются, TUN-адаптер уничтожается.

## Требования

- Windows 10/11
- Права администратора (для создания TUN-адаптера и маршрутов)
- Работающий SOCKS5 прокси (по умолчанию предполагается локальный прокси на порту 10808)

## Установка и запуск

Программа компилируется в один независимый `winchun.exe`. Библиотека `wintun.dll` и утилита `tun2socks.exe` уже зашиты внутрь с помощью `go:embed`.

### 1. Скачивание или Сборка

Для работы на любом ПК вам нужны всего два файла в одной папке:

1. `winchun.exe`
2. `config.json`

Если вы хотите собрать программу из исходников:

1. Установите Go.
2. Скачайте репозиторий.
3. Запустите файл `build.bat` (он выполнит команду `go build -o winchun.exe ./src`).

### 2. Конфигурация

Файл `config.json`:

```json
{
  "socks5": "127.0.0.1:10808",
  "tun_name": "winchun0",
  "tun_addr": "10.0.85.2",
  "tun_gw": "10.0.85.1",
  "tun_mask": "255.255.255.0",
  "dns_refresh_seconds": 120,
  "domains": ["example.com", "*.google.com", "api.openai.com"]
}
```

### Формат доменов

- `example.com` — точное совпадение
- `*.example.com` — резолвит только `example.com` и направляет этот IP в прокси. Внимание: DNS поддоменов с другими IP не будут проксироваться автоматически. Для таких поддоменов прописывайте их явно.

## Запуск

1. Запустите ваш прокси-клиент (Hiddify/v2rayN/Nekobox) и переведите маршрутизацию в режим **Global** (иначе трафик зациклится).
2. Запустите `winchun.exe` **от имени Администратора**.

С кастомным конфигом:

```powershell
.\winchun.exe -config my_config.json
```

## Остановка

Нажмите `Ctrl+C` в консоли. Утилита:

1. Удалит все добавленные маршруты.
2. Остановит tun2socks и удалит виртуальный адаптер.

## Troubleshooting

| Проблема                        | Решение                                                                                    |
| ------------------------------- | ------------------------------------------------------------------------------------------ |
| Ошибка `connectex` (порт занят) | Трафик зациклился. Включите **Global / Proxy** режим маршрутизации в вашем SOCKS5 клиенте. |
| `route add failed`              | Запустите `winchun.exe` от имени администратора.                                           |
| Сайты не открываются            | Проверьте порт SOCKS5 в конфиге и убедитесь, что прокси работает.                          |

## Релизы (GitHub Actions)

В проекте настроен автоматический сборщик релизов через GitHub Actions.
Чтобы создать новый бинарник (`winchun.exe`) и опубликовать его в разделе **Releases** на GitHub, достаточно создать и отправить тег.

```powershell
# 1. Закоммитьте все нужные изменения
git add .
git commit -m "Fix bugs and add features"

# 2. Создайте тег с версией (например, v1.0.0)
git tag v1.0.0

# 3. Отправьте тег на GitHub
git push origin v1.0.0
```

GitHub сам скачает Go, скомпилирует `winchun.exe` и прикрепит его вместе с файлом `config.json.example` к новому релизу.
